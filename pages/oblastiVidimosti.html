Области видимости в JavaScript

Область видимости определяет, где и как переменные и функции могут быть доступны в вашем коде. JavaScript поддерживает глобальную, локальную и блочную области видимости, а также особенности замыканий.

Глобальная область видимости

Переменная объявленная вне всех функций или блоков кода, считается глобальной и доступна из любой части кода.

        var globalVar = "Я глобальная переменная";

        function testScope() {
            return globalVar; // Доступ к глобальной переменной внутри функции
        }

        console.log(testScope()); // "Я глобальная переменная"

Локальная область видимости

Переменные, объявленные внутри функции, находятся в локальной области видимости этой функции и доступны только внутри неё.

        function testLocalScope() {
            var localVar = "Я локальная переменная";
            return localVar;
        }

        console.log(testLocalScope()); // "Я локальная переменная"
        // console.log(localVar); // Ошибка: localVar is not defined

Блочная область видимости

JavaScript (начиная с ES6) поддерживает блочную область видимости, которая позволяет объявлять переменные с помощью let и const в пределах блоков {}.

        function testBlockScope() {
            if (true) {
                let blockVar = "Я переменная в блочной области видимости";
                console.log(blockVar); // Доступна только в этом блоке
            }
            // console.log(blockVar); // Ошибка: blockVar is not defined
        }

        testBlockScope();

Замыкания

Замыкание происходит, когда функция запоминает и продолжает доступ к переменным из своей области видимости, даже после того как она выполнена в месте, где была первоначально объявлена.

        function makeAdder(x) {
            return function(y) {
                return x + y;
            };
        }

        var addFive = makeAdder(5);
        console.log(addFive(3)); // 8
